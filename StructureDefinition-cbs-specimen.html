Resource Profile: Case Based Surveillance Specimen Profile

Defining URL:http://cbsig.chai.gatech.edu/StructureDefinition/cbs-specimen
Version:0.1.0
Name:CaseBasedSurveillanceSpecimen
Title:Case Based Surveillance Specimen Profile
Status:Experimental as of 1/1/21
Definition:

Defines constraints on the Specimen resource in order to meet the needs of public health surveillance programs.

Publisher:Georgia Tech Research Institute (GTRI)
Source Resource:XML / JSON / Turtle

The official URL for this profile is: 

http://cbsig.chai.gatech.edu/StructureDefinition/cbs-specimen

Mandatory and Must Support Data Elements

The following data-elements must always be present (Mandatory definition) or must be supported if the data is present in the sending system (Must Support definition). They are presented below in a simple human-readable explanation. Profile specific guidance and examples are provided as well. The Formal Profile Definition below provides the formal summary, definitions, and terminology requirements. The “Snapshot Table (Must Support)” tab below displays all the mandatory and must support data elements for this profile.

Each Specimen must have:

  1. a patient that conforms to the CBS Patient Profile

Each Specimen must support:

  1. an extension that has a code for the specimen role
  2. a placer assigned ID identifier
  3. a filler assigned ID identifier
  4. a type of specimen
  5. a received time for when the specimen was received by the performing laboratory
  6. a collected date time or period
  7. a collected quantity
  8. a collected body site
  9. a note for any comments on the specimen
