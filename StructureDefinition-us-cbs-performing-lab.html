Resource Profile: US Case Based Surveillance Performing Laboratory Profile

Defining URL:http://cbsig.chai.gatech.edu/StructureDefinition/us-cbs-performing-lab
Version:0.2.0
Name:USCaseBasedSurveillancePerformingLaboratory
Title:US Case Based Surveillance Performing Laboratory Profile
Status:Active as of 2022-02-08 05:12:45-0500
Definition:

Defines further constraints and extensions to the ECR US Public Health Organization profile in order to meet the needs of case based surveillance programs while maintaining adherence to US Core standards.

Publisher:Georgia Tech Research Institute (GTRI)
Source Resource:XML / JSON / Turtle

The official URL for this profile is: 

http://cbsig.chai.gatech.edu/StructureDefinition/us-cbs-performing-lab

Mandatory and Must Support Data Elements

The following data-elements must always be present (Mandatory definition) or must be supported if the data is present in the sending system (Must Support definition). They are presented below in a simple human-readable explanation. Profile specific guidance and examples are provided as well. The Formal Profile Definition below provides the formal summary, definitions, and terminology requirements. The “Snapshot Table (Must Support)” tab below displays all the mandatory and must support data elements for this profile.

Each Organization must have:

  1. an active flag representing whether the record is still in use
  2. a type code of lab
  3. a name
  4. a phone number or data absent reason extension
  5. an address or data absent reason extension

Each Organization must support:

  1. an identifier
